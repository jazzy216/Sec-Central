General Security Concepts

EXAM WEIGHT 12%

CIA Triad + Extensions

Confidentiality — Only authorized parties access data. Tools: encryption, ACLs, steganography, MFA.
Integrity — Data is not tampered with. Tools: hashing (SHA-256), digital signatures, HMAC.
Availability — Systems up when needed. Tools: redundancy, RAID, failover, backups, load balancers.
Extensions
Authenticity — Proves the source. Digital signatures, certificates.
Non-repudiation — Sender can't deny sending. Digital signatures + timestamps + audit logs.
Mnemonic
"CIA + AN" — the triad plus Authenticity & Non-repudiation. Think "CIA Agents Never lie."

AAA Framework

Authentication — Who are you? (passwords, biometrics, tokens)
Authorization — What can you do? (RBAC, ABAC, ACLs)
Accounting — What did you do? (logs, audit trails)
AAA Protocols
  • RADIUS — UDP 1812/1813, encrypts only password, for network access.
  • TACACS+ — TCP 49, encrypts entire payload, granular command authorization, Cisco.
  • Diameter — successor to RADIUS, TCP/SCTP.
  • Kerberos — TCP/UDP 88, uses tickets (TGT → ST), beats replay via timestamps. Requires time sync (within 5 min).

Authentication Factors

  • Something you KNOW — password, PIN, passphrase.
  • Something you HAVE — smart card, token, phone, HOTP/TOTP.
  • Something you ARE — fingerprint, face, retina, vein.
  • Somewhere you ARE — GPS, IP geolocation (attribute).
  • Something you DO — gait, signature dynamics, typing pattern.
MFA vs 2SV

MFA requires factors from different categories. Password + PIN = not MFA (both "know"). Password + SMS code = MFA.

Biometric Metrics
  • FAR (False Accept Rate) — imposter accepted. Security risk.
  • FRR (False Reject Rate) — legit user rejected. Usability risk.
  • CER (Crossover Error Rate) — where FAR = FRR. Lower = better.

Access Control Models

DAC — DiscretionaryOwner decides. Think Windows NTFS permissions. Flexible, weak.
MAC — MandatoryLabels + clearances (Top Secret, Secret…). OS enforces. Gov/military, SELinux.
RBAC — Role-BasedUsers → roles → perms. Easy to manage at scale.
ABAC — Attribute-BasedPolicy on attributes (dept, time, device, location). Most flexible.
Rule-BasedFirewall ACLs, time-of-day rules.

PoLPSeparation of DutiesNeed-to-knowJob Rotation

Zero Trust

Never trust, always verify. Assume breach. Verify every request as if it came from an open network.

Control Plane vs Data Plane
  • Control Plane — Policy Engine (PE), Policy Administrator (PA), Policy Enforcement Point (PEP), adaptive identity, threat scope reduction, implicit trust zones.
  • Data Plane — Subject/System, policy enforcement, the actual traffic flowing.
Key
If the question says "verify every connection" or "micro-segmentation", answer is Zero Trust.

Security Control Types & Categories

By Category (what it IS)
  • Technical — firewalls, encryption, IPS, AV.
  • Managerial — risk assessments, policies, SDLC.
  • Operational — awareness training, backups, incident response.
  • Physical — locks, fences, guards, bollards.
By Function (what it DOES)
  • Preventive — stops it (firewall, ACL).
  • Deterrent — discourages (warning sign, lighting).
  • Detective — finds it (IDS, SIEM, CCTV).
  • Corrective — fixes it (patch, restore).
  • Compensating — substitute (MFA when app can't support SSO).
  • Directive — policy, AUP, "thou shalt."

Change Management

  • CAB — Change Advisory Board approves.
  • Standard / Normal / Emergency change types.
  • Impact analysis, backout plan, test results, maintenance window.
  • Tech impacts: allow/deny lists, restricted activities, downtime, service restart, app restart, legacy apps, dependencies.
  • Documentation: version control, updated diagrams, updated policies/procedures.

FeatureSymmetricAsymmetric

Threats, Vulnerabilities & Mitigations

Threat Actors

  • Nation-state / APT — highly funded, stealthy, long dwell time. Motive: espionage, disruption.
  • Organized crime — financial motive, ransomware gangs.
  • Hacktivist — ideological; doxxing, defacement, DDoS.
  • Insider threat — privileged access, hardest to detect.
  • Script kiddie — low skill, uses existing tools.
  • Shadow IT — unsanctioned cloud/apps, not malicious but risky.
  • Unskilled attacker / Competitor — industrial espionage.

Attack Vectors & Surfaces

  • Message-based — email (phishing), SMS (smishing), voice (vishing).
  • Image-based — malicious SVG/EXIF.
  • File-based — malicious macro docs, ISOs, PDFs.
  • Voice call — vishing, CEO fraud.
  • Removable device — USB drop, BadUSB (HID emulation).
  • Vulnerable software — unpatched OS/app, client-based vs agentless.
  • Unsupported systems — EOL Windows 7, unmanaged IoT.
  • Unsecure networks — open WiFi, rogue AP, Bluetooth, wired access.
  • Open service ports, default creds, supply chain (MSPs, vendors, hardware).

Social Engineering

  • Pretexting — fabricated scenario.
  • Watering hole — compromise a site the target visits.
  • Tailgating / Piggybacking — physical follow-in.
  • Dumpster diving, shoulder surfing, typosquatting.
  • Brand impersonation, misinformation / disinformation.

Application Attacks

  • Injection — SQLi, LDAP, XML, command injection. Mitigation: parameterized queries, input validation.
  • XSS — reflected, stored, DOM. Mitigation: output encoding, CSP.
  • CSRF / XSRF — forces user's browser. Mitigation: anti-CSRF tokens, SameSite cookies.
  • SSRF — server makes attacker-controlled request (cloud metadata stealing).
  • Directory traversal — ../../etc/passwd.
  • Buffer overflow / Integer overflow — stack/heap corruption. Defense: ASLR, DEP, canaries.
  • Race condition / TOCTOU — time-of-check vs time-of-use.
  • Memory leak, null pointer deref, resource exhaustion.
  • API attacks — broken auth, excessive data exposure, lack of rate limiting.
  • Replay, session hijacking, cookie theft.

Cryptographic Attacks

  • Birthday — hash collision; why SHA-1 is dead.
  • Downgrade — forces TLS 1.0 / SSL (POODLE).
  • Collision — two inputs, same hash.
  • Rainbow table — precomputed hash lookup. Defense: salt + key-stretching (bcrypt, PBKDF2, Argon2, scrypt).
  • Pass-the-hash — NTLM hash reuse.
  • Key stretching — slow hashing on purpose to defeat brute force.
  • Side-channel — timing, power, cache (Spectre/Meltdown).
Password attacks

Brute forceDictionaryHybridCredential stuffingPassword spraying

Defense: rate limiting, account lockout, MFA, password managers, long passphrases.

Network Attacks

  • DDoS — volumetric (UDP flood), protocol (SYN flood), application (HTTP GET flood). Defense: scrubbing center, rate limiting, anycast.
  • DNS — cache poisoning, DNS tunneling, domain hijacking, zone transfer (AXFR). Defense: DNSSEC, DoH/DoT.
  • MITM / On-path — ARP poisoning, evil twin, SSL strip. Defense: HSTS, certificate pinning.
  • MAC flooding — fills CAM table → switch acts like hub. Defense: port security.
  • VLAN hopping — double tagging / switch spoofing. Defense: don't trunk unused ports, native VLAN ≠ user VLAN.
  • Rogue AP / Evil twin — defense: 802.1X, WIPS.
  • Wireless — deauth, disassoc, IV (WEP), KRACK (WPA2). Use WPA3-SAE.
  • Bluetooth — bluejacking (send msg), bluesnarfing (pull data), bluebugging (control).
  • RFID/NFC — skimming, replay.

Physical Attacks

  • Brute force — jammed locks, ramming.
  • RFID cloning — copying badges (Proxmark3).
  • Environmental — cutting power, HVAC attacks.
  • Dumpster diving, tailgating, lock picking.
  • Skimmers — card readers.
Physical controls

Mantrap / Access vestibuleBollardsFencingGuardsLightingCCTVBadgesLocks

Vulnerability Management Lifecycle

  1. Identification — scans (Nessus, OpenVAS, Qualys), SAST, DAST, IAST, pen tests, bug bounty, OSINT.
  2. Analysis — confirm, score (CVSS), prioritize. CVE = the ID; CVSS = the score; CWE = the class.
  3. Response — patch, compensating control, risk accept/transfer/avoid, segmentation.
  4. Validation — rescan to verify.
  5. Reporting — KPI/KRI dashboards, exec-level summaries.
CVSS quick

0.0 None · 0.1–3.9 Low · 4.0–6.9 Medium · 7.0–8.9 High · 9.0–10 Critical

Indicators of Compromise (IOCs)

  • Account lockouts / concurrent sessions / impossible travel.
  • Blocked content, published/documented leaks.
  • Resource consumption or inaccessibility (disk fills, CPU spikes).
  • Out-of-cycle logging, missing logs.
  • Unusual DNS / beaconing.
  • New or unsigned processes, persistence (scheduled task, run keys).
Indicators of Attack (IOA)

Behavior-based, future/active — e.g., PowerShell encoded command followed by outbound SMB. Maps to MITRE ATT&CK tactics/techniques.

Security Architecture

EXAM WEIGHT 18%

Deployment Models

  • On-prem — full control, full cost.
  • Cloud — IaaS (VMs/networks), PaaS (runtimes/DBs), SaaS (apps).
  • Public / Private / Community / Hybrid.
  • Shared Responsibility Model — IaaS: you secure OS+. PaaS: you secure apps+data. SaaS: you secure data+identity.
  • Hybrid — cloud + on-prem; worry about consistent IAM + data locality.
  • Centralized vs Decentralized — control vs resilience trade-off.
Virtualization / Containers
  • Type 1 hypervisor — bare metal (ESXi, Hyper-V server). More secure.
  • Type 2 — hosted (VirtualBox, VMware Workstation).
  • VM escape, VM sprawl, snapshot leakage.
  • Containers — shared kernel, smaller attack surface than VMs but kernel escape is catastrophic.

Network Architecture

  • Air-gapped — physically isolated, highest security.
  • Screened subnet (DMZ) — internet-facing servers between two firewalls.
  • Segmentation — VLANs, subnets, security groups; limits lateral movement.
  • Microsegmentation — east/west traffic control, per-workload.
  • SDN — separates control and data plane; programmatic policy.
  • SD-WAN — overlay multiple transports (MPLS/LTE/broadband).
  • SASE — Secure Access Service Edge = SD-WAN + SWG + CASB + ZTNA + FWaaS.
  • NAC — 802.1X posture-checks devices before granting access.

Firewalls & Network Appliances

  • Packet filter — L3/L4, stateless.
  • Stateful — tracks connection state.
  • NGFW — app-aware, user-aware, IDS/IPS, TLS inspection.
  • WAF — L7, HTTP-specific; stops SQLi/XSS.
  • UTM — all-in-one; AV, IPS, URL filter, VPN.
  • IDS (passive) vs IPS (inline, blocks). Signature vs anomaly/heuristic.
  • Proxies — forward (egress), reverse (ingress/load-balancing).
  • Load balancer — active/active vs active/passive.
  • Jump server / bastion — hardened hop for admin access.
  • Honeypot / Honeynet / Honeyfile / Honeytoken — detection via decoy.

Cryptographic Primitives

Symmetric algorithms
  • AES (128/192/256) — FIPS-approved, modern default. Modes: GCM (AEAD, preferred), CBC, CTR, ECB (never!).
  • ChaCha20-Poly1305 — fast on mobile, AEAD.
  • 3DES / DES / RC4 / Blowfish — legacy, avoid.
Asymmetric
  • RSA (2048+ recommended).
  • ECC — shorter keys, same strength (P-256, Curve25519).
  • DH / ECDH / ECDHE — key exchange; ephemeral = Perfect Forward Secrecy.
  • DSA / ECDSA / EdDSA — signatures.
Hashing
  • MD5 (broken), SHA-1 (broken), SHA-2 (256/384/512), SHA-3, BLAKE2/3.
  • HMAC — hash + key → authenticity + integrity.
  • Password hashes: bcrypt, PBKDF2, scrypt, Argon2.

PKI Essentials

  • CA / Root CA / Intermediate / Subordinate — trust chain.
  • CSR — Certificate Signing Request contains public key + subject info.
  • X.509 cert fields: Subject, Issuer, SAN, Validity, Signature.
  • Revocation: CRL (list) vs OCSP (live check) vs OCSP stapling (server provides proof).
  • Pinning — app hard-codes cert/pubkey.
  • Key escrow — third party holds copy for recovery.
  • Wildcard (*.domain.com), SAN (multi-domain), EV, DV, OV.
Cert formats

PEM base64 · DER binary · PFX/P12 private+public · CER/CRT public

Data States & Protection

  • At rest — FDE (BitLocker/FileVault), SED, TPM-bound keys, database TDE.
  • In transit — TLS 1.3, IPsec VPN (AH vs ESP, transport vs tunnel), WireGuard, SSH.
  • In use — harder; trusted execution env (TEE, Intel SGX), homomorphic encryption, enclaves.
Data classification

PublicInternal/PrivateConfidentialCritical/Restricted

Obfuscation techniques
  • Tokenization — replace with non-sensitive token (PCI).
  • Masking — hide part (****-****-****-1234).
  • Hashing — one-way.
  • Anonymization / Pseudonymization — GDPR relevant.

Mobile / Endpoint / IoT / ICS

  • MDM — pushes policy, wipe, app control.
  • BYOD / COPE / CYOD — ownership models.
  • Containerization — work vs personal.
  • Jailbreak/Root — breaks security model.
  • Sideloading — unsigned APKs.
  • IoT — weak defaults, no patching, segment on own VLAN.
  • ICS/SCADA — availability > confidentiality, air-gap where possible, Purdue model, no direct internet.
  • Embedded systems, RTOS, FPGA.

Resilience & High Availability

  • RAID — 0 (stripe, no redundancy), 1 (mirror), 5 (stripe+parity), 6 (double parity), 10 (mirror+stripe).
  • Clustering — active/active vs active/passive.
  • Load balancing — round robin, least connections, weighted, affinity/sticky.
  • Geographic dispersion — multi-region, multi-AZ.
  • Site types: Hot (live, minutes), Warm (hardware + stale data, hours), Cold (empty space, days).
  • Backups — Full / Differential (since last full) / Incremental (since last backup). 3-2-1 rule: 3 copies, 2 media, 1 offsite.
  • Snapshot ≠ backup.
  • Power: UPS, generator, dual PSU, dual power feeds.
Key metrics
RPO = how much data can you lose (time since last backup). RTO = how long to be back up. MTBF = avg uptime. MTTR = avg repair. RPO → past, RTO → future.

Security Operations

EXAM WEIGHT 28%

Baselines & Hardening

  • Secure baseline — documented, enforced config state (CIS Benchmarks, STIGs).
  • Benchmarks — CIS, DISA STIG, NIST checklists.
  • Group Policy, SELinux, AppArmor, AppLocker, WDAC.
  • Disable unused ports/services, change defaults, patch cadence.
  • FDE / TPM / Secure Boot / Measured Boot / UEFI.
  • Host firewall, HIDS/HIPS, EDR, XDR.
  • Wireless hardening — WPA3, 802.1X EAP-TLS, disable WPS.

Identity & Access Management

  • Provisioning / de-provisioning — joiners/movers/leavers.
  • SSO via SAML, OIDC, OAuth 2.0. OAuth = authorization; OIDC = identity layer on top.
  • Federation — trust between identity providers.
  • Privileged Access Management (PAM) — vaulted creds, session recording, JIT access, password rotation.
  • Just-in-time (JIT) + Just-enough-access (JEA).
  • Access reviews, attestations, entitlement reviews.
  • Service accounts — no interactive login, long random passwords, managed (gMSA).
MFA types

HOTPTOTPPushFIDO2/WebAuthnSmart cardSMS (weak)

Logging, Monitoring, SIEM/SOAR

  • Log sources: OS, app, firewall, IDS/IPS, DNS, proxy, AV/EDR, authentication, network flow (NetFlow/IPFIX).
  • Syslog severity 0-7; use a central collector (514 UDP / 6514 TLS).
  • SIEM — aggregates, correlates, alerts. Examples: Splunk, Sentinel, QRadar, ELK.
  • SOAR — playbooks, auto-enrich, auto-contain.
  • UEBA — behavior analytics.
  • Time sync (NTP) is mandatory for correlation.
  • Log retention based on policy/regulation.

Incident Response Lifecycle (NIST SP 800-61)

  1. Preparation — tools, training, IR plan, comms, runbooks.
  2. Detection & Analysis — alerts, triage, scoping.
  3. Containment — short-term (isolate host) & long-term (segment, filter).
  4. Eradication — remove malware, close vector, patch.
  5. Recovery — restore, monitor for return.
  6. Lessons Learned — after-action, update playbooks.
Mnemonic
"Please Don't Call Everyone Right Late." — Preparation, Detection, Containment, Eradication, Recovery, Lessons.

Digital Forensics

  • Order of volatility (RFC 3227): CPU registers/cache → RAM → routing table/ARP/processes → temp files → disk → remote logs → archival media.
  • Chain of custody — who, what, when, where; every transfer logged.
  • Hashing — MD5/SHA-256 before + after; if match, evidence is unchanged.
  • Write blockers for disk imaging.
  • Legal hold — preserve relevant data for litigation.
  • E-discovery.
  • Reporting — clear, reproducible, admissible.
Common tools

FTK ImagerEnCaseAutopsyVolatilityddWireshark

Threat Intelligence & Hunting

  • OSINT, ISACs (sector sharing), STIX/TAXII standard.
  • TTPs — tactics, techniques, procedures. MITRE ATT&CK is the reference.
  • Cyber Kill Chain — Recon → Weaponize → Deliver → Exploit → Install → C2 → Actions on objectives.
  • Diamond Model — adversary, capability, infrastructure, victim.
  • Hunting hypotheses, IoCs vs IoAs.

Network Security Operations

  • Port security, DHCP snooping, Dynamic ARP Inspection, BPDU guard, Root guard, Storm control.
  • 802.1X — supplicant, authenticator, authentication server.
  • VPN — remote access (SSL/TLS, IPsec), site-to-site, split tunnel vs full tunnel, always-on VPN.
  • NAC — posture assessment, quarantine VLAN.
  • DLP — endpoint, network, cloud; based on content (regex, fingerprint), context, classification.

Email Security

  • SPF — TXT record lists allowed senders.
  • DKIM — cryptographic signature of message.
  • DMARC — policy (none/quarantine/reject) + reporting; ties SPF+DKIM to From header.
  • Secure email gateway — sandboxing, URL rewriting, AV.
  • S/MIME, PGP/GPG for end-to-end.
Recall
SPF = who can send. DKIM = was it tampered. DMARC = what to do if checks fail + tell me about it.

Automation & Orchestration

  • IaC — Terraform, ARM, CloudFormation. Reproducible, reviewable, versioned.
  • Config mgmt — Ansible, Puppet, Chef, DSC.
  • CI/CD security — SAST in pipeline, secrets scanning, SBOM generation, signed artifacts.
  • Playbooks / runbooks.
  • Benefits: efficiency, enforcing baselines, reducing error, reacting at machine speed.
  • Risks: complexity, single-point-of-failure if one pipeline owns everything, technical debt.

Awareness & Training

  • Security awareness program — ongoing, not annual-checkbox.
  • Phishing simulations, reporting mechanism (report-phish button).
  • Role-based training — devs vs execs vs helpdesk.
  • Topics: anomalous behavior, insider threat, password hygiene, removable media, reporting, OPSEC, travel.
  • Gamification, metrics (click rate, report rate, dwell-time-before-report).

Crypto Decision Tree

  • Encrypt big data fast → AES-256-GCM.
  • Exchange keys over untrusted → ECDHE (PFS).
  • Sign code/docs → RSA-2048 or EdDSA.
  • Password storage → Argon2 > bcrypt > PBKDF2 (NEVER just SHA).
  • Verify download integrity → SHA-256.
  • Mobile battery-friendly → ChaCha20-Poly1305.
  • Full disk → AES-XTS.
  • Message auth → HMAC-SHA-256.

"Which Attack Is It?" Fingerprints

  • URL has ../ → Directory traversal.
  • Input contains 'OR 1=1 -- → SQLi.
  • <script> in comments → Stored XSS.
  • Server fetches 169.254.169.254 → SSRF.
  • Victim tricked into submitting form elsewhere → CSRF.
  • Many failed logins, one user → Brute force.
  • One password, many users → Password spraying.
  • Leaked creds from another breach → Credential stuffing.
  • DNS answers for sites you didn't request → DNS poisoning/pharming.
  • Open rogue WiFi named like corp SSID → Evil twin.

  • Stop doing the activity → Avoid.
  • Buy insurance or outsource → Transfer.
  • Add controls → Mitigate.
  • Cost > benefit; tolerate it → Accept.
  • Time-boxed waiver → Exemption/Exception.

Exam-Taking Strategy

  • Read the LAST sentence first — it tells you what's actually being asked.
  • Eliminate 2 clearly-wrong options. From the remaining, pick BEST, not "technically-right."
  • When stuck: Confidentiality? Integrity? Availability? — align to CIA.
  • PBQs first vs last? Save PBQs for last if comfortable — they eat time.
  • Mark and move. Don't die on one question.
  • Answer every question — no negative marking.
